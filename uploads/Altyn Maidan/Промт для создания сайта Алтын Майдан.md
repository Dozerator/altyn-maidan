# Мастер-промт для создания сайта «Алтын Майдан»

> Документ подготовлен на основе анализа папки проекта: готовая WordPress-тема (`altyn-maidan`), библиотека UI-ассетов с координатами нарезки (`altyn_maidan_ui_assets`), анкета ТЗ от заказчика (не заполнена), выгрузка данных из группы ВКонтакте (история проекта, фестивали 2010–2026) и фотоархив по годам. Промт написан так, чтобы его можно было целиком передать разработчику или AI-инструменту для завершения/пересборки сайта.

- `assets/js/main.js` (245 строк) — бургер-меню, лайтбокс галереи и т.д.
- `template-parts/` — 9 готовых блоков главной страницы (hero, stats, about, events, gallery, winners, application-form, news, contacts) с реальными PHP-циклами по CPT.
- `altyn_maidan_ui_assets/` — нарезанные картинки из исходного макета с `manifest.json` (координаты) и `README.md` с рекомендациями (иконки/орнаменты доработать в SVG).

Это значит: содержательные решения (цели, аудитория, бюджет, сроки, обязательный функционал) ещё не зафиксированы и их нужно либо уточнить у заказчика, либо принять обоснованные умолчания (см. раздел 10).

## 1. О проекте (факты для копирайта — использовать дословно/близко к тексту)

- Полное название: **Интернациональный культурно-просветительский проект «Алтын Майдан» имени Бисера Кирова**.
- Основан в **2009 году** как площадка для межэтнического диалога.
- Формат: международные инклюзивные фестивали-конкурсы, направленные на сохранение традиционной культуры народов России и мира.
- География: Россия (Крым, Якутия, Северная Осетия — Алания, Абхазия) и зарубежье (Китай).
- Группа ВКонтакте: vk.com/altynmaidan, сфера — Culture, ~1542 участника на момент выгрузки.
- Ключевая ценность бренда — **инклюзивность**: фестивали позиционируются как «инклюзивные», рассчитаны в том числе на людей с ограниченными возможностями (в фотоархиве есть отметки про пляжи и дорожки, приспособленные для колясочников).

## 2. Целевая аудитория

1. Руководители и участники творческих коллективов (фольклорных, хореографических, вокальных, театральных) — ищут фестивали для участия, условия и даты.
2. Педагоги ДШИ, культурные учреждения, региональные министерства/отделы культуры — партнёры и организаторы на местах.
3. Международные партнёры и соорганизаторы (в т.ч. зарубежные — Китай, Абхазия) — им нужна презентационная, статусная подача проекта.
4. Жюри, эксперты, приглашённые гости.
5. Родители и участники с инвалидностью/ОВЗ — инклюзивная составляющая должна быть заметна и не быть формальной галочкой.
6. Журналисты и медиа, освещающие фестивали.

## 3. Цели сайта и целевые действия

- Главное целевое действие: **подать заявку на участие в фестивале** (форма уже спроектирована в `application-form.php`).
- Вторичные действия: подписка на новости, переход в соцсети (ВК), скачивание положения/регламента фестиваля (если появится).
- Сайт должен работать как «витрина доверия» для иностранных и государственных партнёров — подчеркнуть масштаб (15+ лет истории, международный статус, поддержка Россотрудничества и Ассамблеи народов Евразии).

## 4. Технологический стек (уже выбран, не менять без причины)

- CMS: **WordPress**, кастомная тема без билдера (чистый PHP + свой CSS/JS).
- PHP ≥ 8.0, WordPress ≥ 6.0 (см. шапку `style.css`).
- Кастомные типы записей: `event` (Мероприятия, слаг `meropriyatiya`), `winner` (Победители, слаг `nagrady`).
- Таксономии: `event_type` (иерархическая), `event_region` (плоская).
- Мета-поля мероприятия: дата начала/окончания, место проведения, ссылка на заявку.
- Мета-поля победителя: коллектив, номинация, фестиваль, год, место/степень (I/II/III/диплом).
- Кастомайзер: контакты (телефон, email, адрес, VK, Telegram, YouTube) и 4 счётчика цифр на главной.
- Форма заявки: поддерживает Contact Form 7 (если активен), иначе — встроенная HTML-форма с отправкой на `admin-post.php` (обработчик `altyn_submit_application` нужно реализовать/проверить в `functions.php`, если ещё не реализован — добавить обработку, письмо администратору и защиту от спама, например honeypot + nonce, nonce уже есть).
- Google Fonts: **Cormorant Garamond** (заголовки) + **Inter** (текст).

Если требуется полностью статический сайт или другой стек — это решение нужно явно согласовать с заказчиком, так как вся текущая архитектура рассчитана на WordPress.

## 5. Дизайн-система

### Цвета (из `main.css`, переменные `:root`)

- `--color-bg: #FBF7F0` — тёплый кремовый фон.
- `--color-surface: #FFFFFF`.
- `--color-burgundy: #6E1F2B`, `--color-burgundy-dk: #4D121D` — основной акцентный/брендовый цвет (бордовый).
- `--color-gold: #C7A052`, `--color-gold-lt: #E1C98A` — золото, национальный/праздничный орнаментальный акцент.
- `--color-text: #2B2522`, `--color-muted: #746B64`, `--color-border: #E8DED2`.

### Типографика

- Заголовки — `Cormorant Garamond` (serif, торжественный, «этнический/классический» характер).
- Текст — `Inter` (нейтральный sans-serif для читаемости).

### Радиусы/тени/сетка

- `--radius-sm/md/lg: 6/12/20px`.
- `--shadow-card`, `--shadow-btn` — мягкие тени.
- Контейнер: `max-width: 1280px`, адаптивные отступы `clamp(16px, 4vw, 60px)`.

### Визуальный язык

- Восточно-славянский/тюркский орнаментальный декор: золотые линии по углам, арки (`hero-arch-left/right`), волны (`decor-wave-left/right`, `footer-red-wave`) — уже есть готовые PNG/WebP в `assets/images` и повторно нарезаны в `altyn_maidan_ui_assets/03_background_decor`.
- Референс мобильной адаптации — `altyn_maidan_ui_assets/07_mobile_reference/`.
- Общий макет — `maket_ui.png` и два ChatGPT-сгенерированных изображения в корне (визуальные концепты hero/людей в нацкостюмах).

**Рекомендация из README ассетов, которую нужно выполнить:** логотип, орнаменты и иконки сейчас — растровые вырезки из общего макета (с артефактами на полупрозрачных участках). Для продакшена их нужно перерисовать/векторизовать в SVG у дизайнера, а не использовать текущие PNG как финальные.

## 6. Карта сайта

1. **Главная** (`front-page.php`) — hero → цифры проекта → о проекте → мероприятия → галерея → победители → форма заявки → новости → контакты.
2. **О проекте** (`/o-proekte/`) — расширенная версия блока about: полная история 2009–2026, миссия, инклюзивность, партнёры (Россотрудничество, Ассамблея народов Евразии и Африки).
3. **Мероприятия** (`/meropriyatiya/`, архив CPT `event`) — фильтр по типу (`event_type`) и региону (`event_region`).
4. **Карточка мероприятия** (`single-event.php`).
5. **Фестивали / История** — отдельная страница-таймлайн по таблице из раздела 1 (либо реализовать через архив с фильтром по годам).
6. **Награды** (`/nagrady/`, архив CPT `winner`).
7. **Галерея** (`/galereya/`) — расширенная версия блока gallery, с разбивкой по фестивалям/годам (папки фотоархива уже дают готовую структуру альбомов).
8. **Участникам** (`/uchastnikam/`) — условия участия, документы, FAQ (страница упомянута в меню, шаблона пока нет — создать на основе `page.php`).
9. **Заявка на участие** (`/zayavka/`) — отдельная посадочная с формой (сейчас форма встроена в главную, но на неё уже ссылаются кнопки хедера — нужна собственная страница).
10. **Новости** (`/novosti/`, стандартный блог WP).
11. **Контакты** (`/kontakty/`).
12. **Политика конфиденциальности** (`/politika-konfidentsialnosti/`) — уже ссылается форма заявки, страницы физически нет — создать обязательно (юридическое требование для сбора персональных данных).

## 9. SEO и технические требования

- `<title>` управляется WordPress (`add_theme_support('title-tag')`) — подключить SEO-плагин (Yoast/RankMath) для метатегов, OG-разметки, sitemap.
- Изображения — использовать `.webp` там, где уже есть (большинство ассетов подготовлены в webp), `loading="lazy"` уже расставлен на большинстве картинок, hero — `loading="eager"`.
- Structured data: для мероприятий имеет смысл разметка `Event` (schema.org) — дата начала/окончания и место уже есть в мета-полях, можно вывести как JSON-LD.
- Основные размеры изображений уже зарегистрированы через `add_image_size` (`altyn-card` 480×300, `altyn-hero` 1440×700, `altyn-gallery` 600×450, `altyn-square` 400×400) — соблюдать при загрузке новых фото.
- Убрана версия WP из `wp_head` — базовая гигиена безопасности уже сделана, дополнительно стоит закрыть `xmlrpc.php`, ограничить попытки логина, настроить регулярные бэкапы (учитывая профиль владельца — специалиста по ИБ, стоит явно заложить security-хардening WordPress: отключение REST API для гостей там, где не нужно, файрвол уровня приложения, HTTPS/HSTS, минимизация плагинов).

Используя всё вышеизложенное, доработай и заверши сайт «Алтын Майдан» на базе существующей WordPress-темы `altyn-maidan/`:

5. Доработай обработчик формы заявки на сервере (админ-уведомление, антиспам, сообщение об успехе).
6. Замени временные растровые орнаменты/иконки на векторные (SVG) версии у дизайнера.
7. Подключи SEO-плагин, настрой schema.org для мероприятий, проверь Core Web Vitals (lazy-loading уже частично сделан).
8. Закрой базовые пункты безопасности WordPress (список в разделе 9) — это профильная область заказчика, стоит сделать особенно тщательно.
